git pull es para bajar desde la nube a local

git status es para ver si hay algo pendiente de subir

"git add ." es para agregar todos los cambios al área de preparación

"git commit -m 'mensaje'" es para confirmar los cambios con un mensaje

git push es para subir los cambios a la nube

Comandos básicos de Git y su uso:

  • git clone <url>: Crea una copia local de un repositorio remoto usando su URL.
  • git pull: Descarga y fusiona los cambios del repositorio remoto al local.
  • git status: Muestra el estado actual del repositorio, indicando archivos modificados, añadidos o pendientes de confirmar.
  • git add .: Añade todos los archivos modificados al área de preparación para ser confirmados.
  • git commit -m 'mensaje': Guarda los cambios preparados en el historial del repositorio con un mensaje descriptivo.
  • git push: Envía los cambios confirmados desde el repositorio local al remoto.
  • git branch: Lista, crea o elimina ramas en el repositorio.
  • git checkout <rama>: Cambia a otra rama o recupera archivos específicos.
  • git merge <rama>: Fusiona los cambios de una rama en la rama actual.
  • git log --oneline: Muestra el historial de commits en una línea.
  • git restore --source <commit>: Restaura archivos a su estado en un commit específico.
